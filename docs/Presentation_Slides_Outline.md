# 10-Minute Oral Presentation Outline (5 + 5)

## Slide 1: Problem and Vision (30s)
- Domain problem
- Why the API matters
- What is innovative in your approach

## Slide 2: Architecture (45s)
- High-level architecture diagram
- Request flow
- Database model snapshot

## Slide 3: Deliverables Checklist (30s)
- GitHub repo status
- API documentation PDF
- Technical report status
- Slide and demo assets

## Slide 4: API Functionality Demo (75s)
- Auth flow (`register/login/me`)
- CRUD flow (`ingredients`, `recipes`)
- Status codes and validation behavior

## Slide 5: Advanced Features (60s)
- Analytics endpoints
- Security controls
- Why these exceed baseline requirements

## Slide 6: Testing and Reliability (45s)
- Test architecture and execution proof
- Key pass cases and one error scenario

## Slide 7: Version Control and Deployment (30s)
- Commit history quality
- Reproducible setup
- Deployment or demo strategy

## Slide 8: GenAI Usage and Reflection (45s)
- Tools and workflow
- How AI outputs were validated
- What was rejected and why

## Slide 9: Limitations and Future Work (30s)
- Current constraints
- Concrete next improvements

## Slide 10: Q&A Backup (remaining time)
- Trade-off table
- Security considerations
- Alternative architecture options
