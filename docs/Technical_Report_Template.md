# XJCO3011 Coursework 1 Technical Report Template

Use this template to produce your final <=5-page PDF report.

## 1. Project Overview
- Project title
- Problem statement and user value
- Why this domain was selected

## 2. Architecture and Design Choices
- API style choice (REST/GraphQL) and rationale
- Tech stack:
  - Language/runtime
  - Framework
  - Database choice (SQL)
  - Validation/auth/security libraries
- Data model and relationships (include ER diagram)
- API versioning and route design

## 3. Implementation Highlights
- CRUD implementation details
- Authentication flow (access + refresh tokens)
- Error handling and status code strategy
- Security controls:
  - password hashing
  - token revocation/rotation
  - rate limiting
  - input validation
- Analytics or advanced features beyond baseline

## 4. Testing Strategy and Evidence
- Test levels (integration/unit/manual)
- Key scenarios tested
- Representative results and coverage discussion
- Known test limitations

## 5. Version Control and Deployment
- Commit strategy and repository organization
- Branching workflow (if used)
- Run/deploy approach (local and/or hosted)
- Reproducibility details

## 6. Reflection
- Challenges encountered
- How they were solved
- Lessons learned
- What you would improve next

## 7. Limitations and Future Work
- Current constraints
- Performance/security/maintainability improvements
- Potential product extensions

## 8. GenAI Declaration (Required)
For each GenAI tool used:
- Tool name
- Purpose
- Example prompts
- Output usage (direct/adapted/rejected)
- Validation method
- Critical reflection on benefits/risks

Add conversation logs in appendix or external link.

## 9. References
- Datasets
- Libraries/frameworks
- Tutorials/specifications/docs cited
- Use a consistent style (Harvard/IEEE/etc.)
